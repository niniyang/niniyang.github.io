URI、URL、URN的区别

2015-07-06 URL&URI&URN

前言

对URI、URL、URN一直感到很懵，今天各种搜索学习了下，特此记录。

英文版

区别 https://danielmiessler.com/study/url-uri/
概念 https://en.wikipedia.org/wiki/Uniform_Resource_Identifier

先上一张关系图


URI

(Uniform Resource Identifier 的缩写，统一资源标识符)（代表一种标准），用来唯一的标识一个资源。

URN

uniform resource name，统一资源命名，是通过名字来标识资源

URL

(Uniform/Universal Resource Locator 的缩写，统一资源定位符)，它是一种具体的URI，即URL可以用来标识一个资源，而且还指明了如何locate这个资源。

端口

端口(Port)，相当于一种数据的传输通道。用于接受某些数据，然后传输给相应的服务，而电脑将这些数据处理后，再将相应的回复通过开启的端口传给对方。

端口的作用：因为 IP 地址与网络服务的关系是一对多的关系。所以实际上因特网上是通过 IP 地址加上端口号来区分不同的服务的。

端口是通过端口号来标记的，端口号只有整数，范围是从0 到65535。

SUMMARY

copy from this blog

“A URI 可以进一步被分为定位符、名字或两者都是. 术语“Uniform Resource Locator” (URL) 是URI的子集, 除了确定一个资源,还提供一种定位该资源的主要访问机制(如其网络“位置”)。”

“URI可以分为URL,URN或同时具备locators 和names特性的一个东西。URN作用就好像一个人的名字，URL就像一个人的地址。换句话说：URN确定了东西的身份，URL提供了找到它的方式。”

URI是以一种抽象的，高层次概念定义统一资源标识，而URL和URN则是具体的资源标识的方式。

首先，URL、URN是URI的一种（通过那个图就看的出来吧）。但也不是所有的URI都是URL。
让URI能成为URL：“访问机制”，“网络位置”。e.g. http:// or ftp://.。
URN是唯一标识的一部分，就是一个特殊的名字。

总结

下面到了回答问题的时候了：
当我们替代web地址的时候，URI和URL那个更准确？
基于我读的很多的文章，包括RFC，我想说URI更准确。
别急，我有我的理由：
我们经常使用的URI不是严格技术意义上的URL。例如：你需要的文件在files.hp.com. 这是URI，但不是URL–系统可能会对很多协议和端口都做出正确的反应。
你去http://files.hp.com 和ftp://files.hp.com.可能得到完全不同的内容。这种情况可能更加普遍，想想不同谷歌域名上的不同服务啊。

所以，用URI吧，这样你通常技术上是正确的，URL可不一定。最后“URL”这个术语正在被弃用。所以明智吧少年！
